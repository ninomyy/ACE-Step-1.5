# ⚙️ AutoComposer 技術仕様書 (Technical Spec)

本ドキュメントは、AutoComposerの裏側で動いているシステムアーキテクチャ、設計思想、およびパラメータの決定根拠について、開発者（および将来のメンテナンス担当者）向けに解説するものです。

## 1. システムアーキテクチャ

AutoComposerは以下の3つのレイヤーで構成されています。

1. **GUI (AppleScript)**: `AutoComposer.app`
   - ユーザーとの対話窓口。
   - 動作モード（new/reproduce/random）、モデル（sft/turbo）、ビット深度（16/24/32）を受け取り、シェルスクリプトに引数として渡します。
2. **Launcher (Bash)**: `run_autochord_desktop.sh`
   - Python仮想環境（.venv）をアクティベートし、OllamaとDiTを統合したPythonスクリプトを呼び出します。
3. **Backend (Python)**: `generate_advanced_song.py`
   - Ollama APIと通信して作詞・作曲プロンプトを生成。
   - Ollamaのメモリを解放（VRAMの確保）。
   - ACE-Step 1.5 (DiT) パイプラインを呼び出し、音声合成を実行。

---

## 2. AIプロンプトの設計思想（アプローチF）

ACE-Step 1.5（特にOllama 32Bモデルとの組み合わせ）において、最も高品質な楽曲を安定して生成するためのプロンプト設計を「アプローチF」と定義しています。

### 1. キャプションの長さ制限 (30〜50語)
ACE-StepのT5テキストエンコーダーは、長すぎる文章を与えると重要なキーワード（楽器やジャンルなど）のAttentionが希釈され、結果的に「ぼやけた、特徴のない曲」になりがちです。
そのため、Ollamaへのシステムプロンプトにおいて**「必ず30〜50語の簡潔なカンマ区切りで出力すること」**を厳命しています。

### 2. エンハンスAIの意図的なバイパス
本来のACE-Stepのパイプラインには、ユーザーの短い入力を長く装飾する「エンハンスAI（`format_sample_from_input`）」が組み込まれていました。
しかし、このエンハンスAIが「with synth pads and lingering chords」といった不要な装飾語（Synth Pad汚染）を強制付与し、結果として「D#の音が鳴り止まない」などの音響的バグを引き起こす原因となっていました。
AutoComposerでは、**Ollamaが生成した高品質な英語プロンプトを、エンハンスAIをバイパスして直接DiTに流し込む設計**を採用しています。

## 3. メロディー再現機能（Seed固定）の裏側

「再生成モード（アレンジ固定）」は、楽曲のメロディー骨格とアレンジを完全に維持したまま、歌詞のひらがな（アライメント）のみを修正するための機能です。

### 動作原理
1. `generate_advanced_song.py` が起動した際、`action_mode == "reproduce"` であれば、指定された歌詞ファイル（`Lyrics_...txt`）と同じベース名を持つメタデータ（`Song_...json`）をデスクトップから探索します。
2. JSONをパースし、前回の生成で使用された `seed` の数値を抽出します。
3. `GenerationParams` を構築する際、この抽出したシード値を渡します。
4. DiT（Diffusion Transformer）は決定論的な性質を持つため、**「同じプロンプト」「同じシード」「同じSteps/Shift」**を与えられれば、物理シミュレーションのように前回と全く同じ波形（メロディー・アレンジ）を生成します。テキスト（歌詞）の変更分だけが、内部のクロスアテンションによってボーカルの発音リズムの調整として適用されます。

## 4. モデル別パラメータ設計

Playgroundのデフォルト値および検証結果に基づき、各モデルのパラメータは以下のようにハードコードされています。

### SFTモデル (高解像度・高品質用)
* `inference_steps`: 50
* `shift`: 3.0
* `dcw_enabled`: False
* `guidance_scale`: 7.0

### Turboモデル (高速生成用)
* `inference_steps`: 8
* `shift`: 3.0（※重要: 1.0にするとボーカルが高周波ロスを起こし不鮮明になるため3.0に固定）
* `dcw_enabled`: True
* `guidance_scale`: 7.0（※TurboのDiT内部で自動的に無視されるため、値は何でも無害）

---

## 5. オーディオビット深度のルーティング

DAW（Logic Pro等）でのミックスダウンにおいて、高解像度の恩恵を最大限に受けるためのフォーマットルーティングを実装しています。
* **16bit**: `FLAC` 形式（デフォルト・容量節約）
* **24bit / 32bit Float**: `WAV` 形式（DAWでのクリッピング耐性と互換性重視）
